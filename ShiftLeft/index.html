🔒 Shift Left Security Demo

Interactive Software Supply Chain Security Visualization

🎯 Activity Instructions

This visualization demonstrates how security vulnerabilities hide in software dependencies. Follow these steps to explore supply chain security:

  1. Observe the network: Each circle represents a software package your project depends on
  2. Run Security Scan: Click the blue button to simulate npm audit
  3. Analyze Results: Red/orange circles show vulnerable packages
  4. Toggle View: Hide/show vulnerabilities to see the difference

⚠️ Vulnerable Packages in This Demo

This demo includes intentionally outdated packages with known vulnerabilities:

  • axios v0.21.1 - Critical: Server-Side Request Forgery
  • lodash v4.17.20 - High: Prototype Pollution
  • moment v2.29.1 - High: Regular Expression Denial of Service
  • express v4.17.1 - Moderate: Various middleware vulnerabilities
# Commands to try in your terminal: npm audit # Scan for vulnerabilities npm audit fix # Automatically fix issues npm list --depth=2 # Show dependency tree npm why axios # See why a package is installed npm outdated # Check for updates

🛡️ Security Best Practices

  • Run npm audit regularly in your projects
  • Keep dependencies updated, especially security patches
  • Review what packages you're adding to your project
  • Use automated tools like Dependabot for monitoring
  • Understand that every dependency is code you're trusting

Learning Goals: Understand software supply chain risks • Use security scanning tools • Practice vulnerability remediation • Implement "shift left" security practices
